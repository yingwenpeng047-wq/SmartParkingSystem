\# 智慧停车管理系统



&#x20; 基于 \*\*C++ / Linux\*\* 的高并发停车场管理服务器，采用前后端分离架构。



&#x20; ## 架构



&#x20; 客户端 ←TCP→ 前置服务器(Epoll+线程池) ←共享内存+信号量+消息队列→ 后置服务器(线程池+MySQL)



&#x20; - \*\*前置服务器\*\*：C++ Epoll 边缘触发，管理客户端连接、心跳检测(300s超时)、CRC32 校验、请求转发

&#x20; - \*\*后置服务器\*\*：C++ 工厂模式+单例，处理登录注册、车辆出入场、视频/图片分片上传，对接 MySQL

&#x20; - \*\*进程间通信\*\*：System V 共享内存(环形缓冲区100块) + 信号量(每块独立锁) + 消息队列(通知索引)



&#x20; ## 功能



&#x20; - 用户注册、短信验证码登录

&#x20; - 车辆入场/出场记录、费用核算

&#x20; - 图片/视频分片上传与自动合并

&#x20; - 历史记录分页查询

&#x20; - 视频录像管理

&#x20; - 操作日志记录



&#x20; ## 技术栈



&#x20; | 技术 | 说明 |

&#x20; |------|------|

&#x20; | Epoll (ET) | 高并发 TCP 网络 I/O |

&#x20; | POSIX pthread | 线程池（2-8线程，生产者-消费者模型） |

&#x20; | System V IPC | 共享内存 + 信号量 + 消息队列 |

&#x20; | CRC32 | 自定义二进制协议数据校验 |

&#x20; | MySQL Connector/C++ | 数据库操作（预编译语句防注入） |

&#x20; | Visual Studio | 开发环境，目标平台 Linux |



&#x20; ## 目录结构



&#x20; | 目录 | 说明 |

&#x20; |------|------|

&#x20; | `ParkSystem\_FrontServer/` | 前置服务器（TCP 接入层） |

&#x20; | `ParkSystem\_BackServer/` | 后置服务器（业务逻辑 + 数据访问） |



&#x20; ## 协议



&#x20; 自定义二进制协议：`HEAD(业务类型+包长度+CRC32)` + 结构体包体，定义在 `protocol.h`（两端一致），支持 16 种消息类型。



&#x20; ## 编译运行



&#x20; ### 依赖

&#x20; - MySQL 8.0+

&#x20; - MySQL Connector/C++

&#x20; - pthread



&#x20; ### 构建

&#x20; 1. 用 Visual Studio 打开 `ParkSystem\_BackServer/ParkSystem\_BackServer.sln`

&#x20; 2. 目标平台选择 Linux，配置远程编译

&#x20; 3. 执行 `schema.sql` 初始化数据库

&#x20; 4. 先启动后置服务器，再启动前置服务器（监听 10001 端口）




 ## 客户端

  基于 Qt 5.9 的桌面客户端，提供完整的停车场管理界面。

  - 实时视频监控展示
  - OpenCV 图像采集 + EasyPR 车牌识别
  - TCP 长连接与服务器通信，自定义协议粘包处理
  - 图片/视频上传、车辆出入场记录、历史查询

  技术栈：Qt 5.9 / OpenCV / EasyPR / TCP Socket / 多线程并行
